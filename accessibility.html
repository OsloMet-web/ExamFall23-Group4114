Tilgjengelighetstest

For å sikre kvaliteten på vårt nettsted bruker vi Lighthouse, en tilgjengelighetstestutvidelse for Google Chrome.

Hvordan fungerer Lighthouse

Dette verktøyet hjelper oss med å oppdage mulige problemer og sørge for at nettstedet vårt forbedrer kvaliteten. Med Lighthouse gjennomgår vi nettstedets struktur, innhold og funksjonalitet med disse vurderingskriteriene:

  • Performance: Måler ytelse og finner muligheter for å øke hastigheten på sideinnlastingen.
  • Accessiblity: Om alle kan få tilgang til innhold og navigere på nettstedet ditt effektivt.
  • Best Practices: de beste fremgangsmåtene under skriving av koden.
  • SEO: Search engine optimalisation. Hvor godt kan søkemotorer forstå innholdet ditt.
  • PWA: Mål om nettstedet ditt er raskt, pålitelig og installerbart som en progressive web app.

Derfra kan du bruke vurderingene som veiledning til å forbedre siden. Du vil motta et dokument som forteller hva er galt, samt hvordan det kan fikses.

Vi tester alle html filene våre men viser kun index.html i denne rapporten.

Tilgjengelighet

Desktop - 100/100

Relevante feil: ingen.

Design-valg: For å opprettholde tilgjengelighet for alle brukere, inkl de med skjerm-lesere, har vi:

  • Brukt "alt" og "aria-label" på alle bilder og relevante elementer.
  • Brukt semantic elements der mulig.
  • Opprettholdet kontrast standarer for å gjøre siden lettlest.
  • Optimalisert til å skalere seg, uansett skjermstørrelse, der alt er synlig.

SEO

Desktop - 100/100

Design-valg:

  • Brukt meta description for å forklare hva nettsiden går ut på, og en meta viewport som skalerer nettsiden til enheten.
  • Brukt title element for tittel.
  • Gitt forklaringstekster til linker.
  • Brukt semantic elements der mulig, f.eks nav, header, main, footer, etc.
  • Brukt heading elementer
  • Bilder har alt descriptions.
  • Optimalisert innlastnings-hastigheten på nettsiden.
